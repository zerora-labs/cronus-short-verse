# CronusShortVerse

**叙事引力驱动的短剧宇宙**

CronusShortVerse 是一个开放、由社区驱动的框架，用于构建相互连接的短剧宇宙。它借助天体物理隐喻（恒星、行星、彗星、流星、黑洞）来建模角色、关系和剧情动态，帮助创作者设计快节奏、强戏剧性的故事，牢牢抓住观众注意力。

---

## 什么是短剧？

在 CronusShortVerse 的语境中，**短剧** 指的是每集时长约为 **2 分钟** 的视频系列。这类剧集具备以下特征：
- **快节奏** – 场景切换迅速
- **高强度冲突** – 每一刻都充满张力
- **频繁的悬念** – 迫使观众观看下一集
- **病毒式钩子** – 专为短视频平台（如抖音、Reels、YouTube Shorts）设计

短剧是一种独特的叙事形式，要求高效、冲击力和连续吸引力。

---

## 愿景：短剧的共享宇宙

大多数短剧都是独立作品。CronusShortVerse 改变了这一点，提供了 **共享的叙事基础设施**：

- **跨剧连接** – 不同系列的角色可以互动
- **不断演化的设定** – 一个剧中的事件会影响整个宇宙
- **创作者协作** – 多位作者可以为同一个世界贡献内容

你可以把它想象成 **短剧领域的漫威电影宇宙**，但是开放的、由社区共建的。

---

## 核心功能

- **角色书与世界书**  
  用户可提供角色设定（性格、目标、背景）和世界设定（历史、法则、地理）。系统也支持输出演化后的更新版本。

- **社区投票演化**  
  参与者对角色弧光、重大事件、世界规则变更进行提案和投票。投票结果直接更新角色书/世界书，实现去中心化的集体叙事。

- **AI 创作演化**  
  集成大语言模型生成故事片段、对话或事件分支。用户可挑选、修改并采纳 AI 生成内容，将其纳入宇宙历史。

- **历史可追溯**  
  每一次演化（投票或 AI）都被记录。用户可回溯或回滚到角色或世界的任意历史版本。

## 核心隐喻：天体

我们将每个叙事元素建模为一种天体。这让复杂的关系变得可视且直观。

| 天体 | 叙事角色 | 示例 |
| -------- | ------------------- | ---------------------- |
| **恒星** | 核心主角 / 中心人物 | 商战短剧中的 CEO |
| **行星** | 常驻配角 | 忠诚的助理、主要对手 |
| **彗星** | 客串角色 / 临时影响 | 路过的不速之客 |
| **流星** | 一次性事件 / 噪音 | 一个谣言、一场意外 |
| **黑洞** | 高风险点 / 未知 | 一个可能摧毁一切的秘密 |

**引力场** 代表外部力量（市场趋势、社会压力、反派行为）对角色轨道的拉扯。

## 与 CronusCycle 的关系

CronusShortVerse 构建在 **CronusCycle** 之上。CronusCycle 是一个开源决策审计中间件，它提供：
- **噪声过滤** – 从故事数据中分离信号与噪声
- **逻辑审计** – 确保角色行为与其设定一致
- **AI 插槽** – 可接入任意大语言模型辅助生成故事
- **星系引擎** – 可视化模块，将你的故事宇宙渲染为交互式星图

因此，CronusShortVerse 不仅是一个概念，更是一个**可运行、可扩展的创作者工具**。

## 路线图

我们不只想写文档，更想打造一个**真正可用的平台**。以下是规划：

- **2026 年第三季度 – 网页原型**  
  提供一个简单的网页界面，支持：
  - 创建或导入角色书、世界书
  - 使用星系引擎可视化你的宇宙（基础 2D 星图）
  - 提交演化提案、参与社区投票

- **2026 年第四季度 – AI 集成**  
  一键 AI 生成故事分支，用户可直接采纳 AI 建议并更新角色书/世界书。

- **2027 年第一季度 – 移动 App（iOS 和 Android）**  
  配套 App，支持浏览宇宙、接收投票结果通知、随时快速生成故事。

- **更远的未来** – 开放 API、自定义可视化主题、去中心化治理实验。
